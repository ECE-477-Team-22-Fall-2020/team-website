As we have collected most of the parts for our project, we are now testing individual elements of our design. Due to my being physically away from the rest of the team, I am responsible for handling most of the software design. To work on this, I've begun developing the Android application we will use to communicate with our board's microcontroller and the web server.

For this, the first phase of development is to create Bluetooth functionality to pair our mobile device with the board. A basic outline of this process is detailed in the figure below. I am using Android Studio for application development. Essentially, our application will use the Android Bluetooth API to search for all nearby Bluetooth devices. The HC-05 Bluetooth module we are using will at this point display its name and MAC address. By default, the module's password is 1234. We will not change this and instead will have the application automatically input this password. Once paired, the module can be instructed with data from the application. My efforts for this week were focused on developing the mobile Android application that will be used for our application. The current functionality of the application allows the following process: (1) listing available nearby Bluetooth devices, (2) pairing the mobile device with the Bluetooth device, and (3) sending data in byte packages to the Bluetooth device. The data we send will be related to board logic and will be interpreted by the microcontroller.
